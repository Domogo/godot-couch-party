# Godot Asset Store submission

Godot Couch Party is prepared as a free `Addon` submission for the Godot Asset Store. The store is currently in beta, so recheck the official submission documentation before uploading a new version.

## Listing copy

**Name:** Godot Couch Party

**Summary:** Reusable local multiplayer roster, isolated controller input, and six-slot lobby for Godot 4.

**Description:**

Godot Couch Party supplies the infrastructure shared by local multiplayer games: stable join and reconnect behavior, per-controller input isolation, ready state, human and bot roster slots, Easy/Medium/Hard bot metadata, and a reusable lobby for up to six participants. The runtime-only GDScript addon stays independent of game rules, character spawning, scoring, and bot intelligence. Copy the addon folder into a Godot 4.7 project and adapt its semantic input frames and roster snapshots to your game.

- **Type:** Addon
- **License:** MIT
- **Minimum Godot version:** 4.7
- **Tags:** GDScript, Local Multiplayer, Controller, Input, Lobby, Party, UI

## Release checklist

- Run `./tools/verify.sh` with the pinned Godot 4.7 stable build.
- Run `./tools/package.sh` and inspect the archive contents.
- Confirm `LICENSE` and the store license both say MIT.
- Confirm the source URL points at the public repository.
- Upload `media/couch_party_lobby.png` as the 16:9 screenshot.
- Attach the packaged ZIP to the matching GitHub release.
- Copy the changelog entry for the submitted version.
- Include the AI usage disclosure above.
- Test the downloaded release ZIP in a clean Godot project before submitting for review.

The official requirements include a working asset, a proper `.gitignore`, no essential submodules, a repository license file with copyright ownership, proper English listing text, and a direct icon link. See Godot's [submission guide](https://docs.godotengine.org/en/stable/community/asset_store/submitting_to_asset_store.html) and [Asset Store FAQ](https://docs.godotengine.org/en/stable/community/asset_store/faq.html).
